# 🔨 QFORGE 설계 장점 — 이지 설명 (ASCII + 비유)

> 자체 제일원리 el-ph 엔진 QFORGE("양자 대장간")의 설계 장점.
> ※ 장점 1(무중단 점진 교체)·2(상위 절반 조립)는 QFORGE.md/DESIGN.md 본문에 이미 기록 — 여기는 3~6.

---

## 3. 🔺 3중 정렬 (층 = 검증 = 기억) — "삼각대"

각 층 출력 = `hexa verify` g5 = atlas 노드. 셋이 한 점에 묶임.

```
        층(코드)
         ╱╲
        ╱  ╲
   verify──atlas
  (g5 게이트) (회귀DB)
  → 코드 바뀌면 자동 회귀 + "수렴 안 한 값" 원천봉쇄(d6 기계화)
```

> 비유: 카메라 삼각대 — 세 다리가 서로 잡아줘 안 흔들림(거짓 결과가 비집고 못 듦).

---

## 4. ♻️ atlas 캐시 / fleet warm-start — "이웃집 불씨"

한 후보의 수렴 결과를 atlas에 저장 → 이웃 조성·압력 후보의 출발점으로 재사용.

```
LaH10 수렴ρ ──fold──> atlas
                       │ warm-start
        YH10·CeH9 ────┘  (SCF 반복 ↓↓)
```

> 비유: 옆집에서 불씨 얻어오기 — 매번 부싯돌부터 안 켬.

---

## 5. 📐 미분 가능 (autograd 역설계) — "온도조절 다이얼"

autograd로 ∂Tc/∂(구조·압력) 해석미분 → "Tc 올리려면 뭘 바꾸나" 역방향.

```
정방향:  구조 → [엔진] → Tc          (QE: 여기까지만)
역방향:  Tc↑ 원함 → ∂Tc/∂구조 → 어디 비틀지   (QFORGE 추가)
```

> 비유: 결과 보고 끝(QE) ↔ 원하는 온도 다이얼 돌리면 설정이 따라옴(QFORGE).

---

## 6. 🏠 hexa-native 자기소유 — "내 집 부엌"

외부 Fortran 의존 0. AI가 직접 오케스트레이션·검증·캐시.

```
QE 노선:   레시피 → [남의 식당] → 결과 받아옴 (블랙박스)
QFORGE:    원소 → [내 대장간 전 구간] → 검증·기억·재사용
```

> 비유: 남의 식당 주문(QE) ↔ 내 부엌에서 재료부터(QFORGE) — 모든 단계 손에 쥠.

---
