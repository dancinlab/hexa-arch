## §0. 방법론 — value-of-information (VoI)

**known-unknown** = 현 시점 미관측이지만 그 값에 따라 결론(grade · thesis 방향)이 바뀌는 변수. 이미 닫힌 것(🟢 recompute 재현 · 🔴 deterministic disagreement)이나 본질적 영구 ⚪(가치판단·metaphor·enum)는 제외 — VoI ≈ 0 이기 때문이다.

### 0.1 known-unknown 포함 기준

```
포함  =  (현 tier ∈ {⚪-일시, 🟠, weakened})  ∧  (도착 가능한 단일 측정이 존재)  ∧  (그 측정이 grade/thesis 를 분기)
제외  =  (영구 ⚪ — 가치판단/metaphor/enum, verify N/A by design)
      ∨  (이미 🟢 recompute 재현 완료 — 산술층은 닫힘)
      ∨  (🔴 closed negative — 이미 deterministic disagreement)
```

→ X4 §4.3 의 "일시 ⚪ ~17" + X8 의 🟠 45 + X13 의 weakened 4 / 🔴-flip 1 / 숨은 🔴 후보 6 이 본 레지스트리의 모집단. 영구 ⚪ ~14 (NRF 가중치/metaphor 6 · ISR enum 3 · NRF prognostic 외부행동)는 **VoI ≈ 0** 으로 §5 에 별도 분리(측정해도 결론 안 바뀜).

### 0.2 VoI 우선순위 = 영향력 × 측정 실현성

```
VoI 등급 (5-tier)
  ★★★★★  thesis 전체를 gate (한 측정이 메타도메인 핵심 인과를 결정)
  ★★★★   도메인 핵심 claim 을 🟢/🔴 로 분기 + 측정 실현성 높음
  ★★★    단일 claim escalation + 측정 readout 예정
  ★★     범위/대표성 보강 (산술층은 이미 견고)
  ★      측정은 가능하나 영향 한정 (이미 fenced 거나 견고)

영향력  = 분기되는 claim 수 + thesis 근접도 (X13 ★ 수 carry)
실현성  = 즉시 가능(0m) > readout 예정(일정 확정) > 장기(2-3y+) > 인프라 의존
```

### 0.3 g5 / d6 안전장치

- **g5**: 본 X14 의 "현 tier" 열은 X8 ledger VERBATIM · X13 weakened/🔴-flip 라벨 VERBATIM. **X14 가 새 grade 부여 0건** — known-unknown 식별과 측정 매핑만.
- **d6 (측정 전엔 모른다 정직)**: "resolve되는 것" 열은 **어느 방향으로 분기할지 미리 단정하지 않는다** — 양방향(🟢-positive / 🔴-null)을 모두 명시. 측정 전 결과를 가정하면 VoI 정의 자체가 무너진다.
- **d2**: 측정 불가/장기 영역도 대안 path 명시 (§4).
